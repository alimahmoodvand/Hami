تکنولوژی ها و طراحی های بکار رفته در محصول:

# فرانت
IONIC

# بک
### دیتابیس
MONGO
### سرور
NODEJS

<p dir='rtl' align='right'>
تصمیم تیم ما برای طراحی فنی سیستم، بهره گیری از api و جداسازی فرانت و بک از یکدیگر جهت بهبود سرعت و تمیزی کد و همچنین افزایش قابل توجه قابلیت نگهداری کد بوده است. برای تکنولوژی فرانت تصمیم بر استفاده از cross platform ها شد تا بتوانیم با داشتن code base واحد چند خروجی یکپارچه بگیریم تا به افزایش سرعت و maintainability بالاتری برسیم.
سپس انتخاب بین دو تکنولوژی محبوب تر cross در community یعنی ionic و react مطرح شد که پس از بررسی دلیل محکم تر برای انتخاب منابع انسانی و حضور نیروی متخصص تیم بود که دو نفر به ionic تسلط داشتند.
</p>


<p dir='rtl' align='right'>
در بحث انتخاب تکنولوژی backend هم با توجه به hackathon بودن رویداد و نیاز به تغییرات سریع و احتمالا زیاد، بهترین نوع دیتابیس را NoSQL تشخیص دادیم که انتخاب MONGO صورت گرفت و در مورد زبان برنامه نویسی هم به دلیل concurrent بودن و perfectly match بودن node با mongo و همچنین قابلیت develope سریع نیروی backend تیم در node، این تکنولوژی ها نیز برای backend انتخاب شد.
</p>


<p dir='rtl' align='right'>
همچنین برای بحث readability, maintainability, clean coding, good practice از mvc design pattern هم در api نویسی backend و هم در ionic بهره بردیم.
</p>
